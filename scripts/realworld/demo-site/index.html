Mara

AI-native log shipper and telemetry agent for AI agents and LLM workloads — Rust, OpenTelemetry gen_ai.*, Apache 2.0.

What it captures Where it ships Design Ollama

What Mara captures

  • Tier A: OpenTelemetry receive — Claude Code, Codex, Gemini CLI.
  • Tier B: JSONL / hooks — Cursor Agents, Kimi; redundant signal for ZDR.
  • Tier C: Analytics REST — Augment Code.
  • Proxy tier: HTTP reverse proxy for runtimes without native telemetry — Ollama.

MVP focus: Claude Code (Tier A) and Ollama (proxy) end-to-end.

Where Mara ships

OTLP (gRPC + HTTP), Loki, Splunk HEC, Elasticsearch, object stores (JSONL + Parquet), Kafka, Prometheus remote write, local files, stdout, webhooks.

Design pillars

  • Edge-first single binary; optional gateway (v2); optional control plane (v3).
  • Canonical schema aligned with OpenTelemetry gen_ai.*.
  • WASM-sandboxed policy with PII/PHI/PCI packs.
  • Zero phone-home; raw-body capture opt-in; ZDR-respecting.

Ollama proxy

Point clients at Mara’s listen address instead of :11434. Mara forwards to upstream Ollama, normalizes completions to canonical events (model, usage, durations), and writes JSONL or other sinks.

Cloud models (e.g. gpt-oss:120b-cloud) use the same API; telemetry shape matches local runs.
